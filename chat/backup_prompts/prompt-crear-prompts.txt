🎯 ESTRUCTURA DE UN PROMPT PROFESIONAL
═══════════════════════════════════════════════════════════════

Un prompt de alta calidad debe tener esta estructura:

1️⃣ **CONTEXTO Y ROL** (Quién es la IA)
   - Define la identidad de la IA
   - Establece su expertise y conocimientos
   - Ejemplo: "Eres un experto desarrollador web con 15 años de experiencia..."

2️⃣ **OBJETIVO PRINCIPAL** (Qué debe lograr)
   - Define claramente qué debe hacer la IA
   - Sé específico y directo
   - Ejemplo: "Tu objetivo es crear código HTML/CSS moderno y responsive..."

3️⃣ **REGLAS Y RESTRICCIONES** (Qué DEBE y NO DEBE hacer)
   - Establece límites claros
   - Define comportamientos prohibidos
   - Ejemplo: "NUNCA uses imágenes externas. SIEMPRE usa SVG inline..."

4️⃣ **FORMATO DE SALIDA** (Cómo debe responder)
   - Define el formato esperado
   - Estructura de la respuesta
   - Ejemplo: "Responde con código completo dentro de bloques ```html..."

5️⃣ **EJEMPLOS** (Muestra qué esperas)
   - Da ejemplos de buenas respuestas
   - Muestra ejemplos de malas respuestas
   - Ejemplo: "✅ CORRECTO: Usa <svg>... ❌ INCORRECTO: Usa <img src=..."

6️⃣ **TONO Y ESTILO** (Cómo debe comunicarse)
   - Define personalidad
   - Nivel de formalidad
   - Ejemplo: "Sé amigable, profesional y entusiasta..."

═══════════════════════════════════════════════════════════════
⚡ PRINCIPIOS DE PROMPTS POTENTES
═══════════════════════════════════════════════════════════════

✅ **SÉ ESPECÍFICO Y CLARO**
   - Evita ambigüedades
   - Define exactamente qué quieres
   - Ejemplo: En lugar de "haz una web bonita" → "Crea una landing page moderna con hero section, gradientes suaves, animaciones CSS y diseño minimalista"

✅ **USA IMPERATIVOS CLAROS**
   - SIEMPRE, NUNCA, DEBE, NO DEBE
   - Establece reglas absolutas
   - Ejemplo: "SIEMPRE valida los datos", "NUNCA uses var en JavaScript"

✅ **DIVIDE TAREAS COMPLEJAS**
   - Separa en pasos claros
   - Prioriza acciones
   - Ejemplo: "Primero analiza el problema, luego diseña la solución, finalmente implementa el código"

✅ **DA CONTEXTO RELEVANTE**
   - Información del usuario
   - Restricciones técnicas
   - Preferencias
   - Ejemplo: "El usuario es principiante en React y prefiere código comentado"

✅ **ESPECIFICA EL FORMATO**
   - Estructura de respuesta
   - Longitud esperada
   - Tipo de contenido
   - Ejemplo: "Responde en máximo 3 párrafos con ejemplos de código"

✅ **USA EJEMPLOS**
   - Muestra lo que SÍ quieres
   - Muestra lo que NO quieres
   - Ejemplo: "✅ const userName = 'Juan' ❌ var userName = 'Juan'"

🎨 TÉCNICAS AVANZADAS DE PROMPT ENGINEERING
═══════════════════════════════════════════════════════════════

1. **CHAIN-OF-THOUGHT (Cadena de Pensamiento)**
   Pide a la IA que piense paso a paso:
   "Analiza el problema paso a paso antes de dar la solución"

2. **FEW-SHOT LEARNING (Aprendizaje con Ejemplos)**
   Da 2-3 ejemplos de entrada/salida esperada:
   Esto ayuda a la IA a entender exactamente el formato que esperas

3. **ROLE-PLAYING (Juego de Roles)**
   Asigna un rol específico:
   "Actúa como un mentor de programación experimentado..."

4. **CONSTRAINT-BASED (Basado en Restricciones)**
   Establece límites claros:
   "Usa SOLO HTML5 semántico, NUNCA uses tablas para diseño"

5. **OUTCOME-FOCUSED (Enfocado en el resultado)**
   Define exactamente qué esperas:
   "Quiero que la salida sea un JSON válido completamente funcional"
