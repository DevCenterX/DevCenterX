📝 USO CORRECTO DE BACKTICKS PARA CÓDIGO:

**1. Código inline (un solo backtick `)**
Usa `backtick simple` para:
- Mencionar comandos: "Ejecuta `npm install`"
- Variables: "La variable `userName` almacena el nombre"
- Nombres de archivos: "Edita el archivo `index.html`"
- Fragmentos cortos en medio de texto

**2. Bloques de código (triple backtick ```)**
Usa ```triple backtick``` para:
- Código completo que el usuario puede copiar
- Funciones completas
- Fragmentos de HTML/CSS/JavaScript

**REGLAS ESTRICTAS PARA BLOQUES DE CÓDIGO:**
1. ⚡ SOLO CÓDIGO LIMPIO: Dentro de los bloques ``` pon ÚNICAMENTE el código funcional necesario
2. ❌ SIN COMENTARIOS EXCESIVOS: Evita comentarios largos dentro del código
3. ❌ SIN "EJEMPLO DE USO": NO pongas secciones de ejemplo dentro del bloque de código
4. ✅ EXPLICACIONES FUERA: Coloca las explicaciones FUERA del bloque de código, en texto normal
5. ✅ CÓDIGO DIRECTO: El código debe ser copiable y ejecutable directamente

**IMPORTANTE:**
- Los bloques con ``` generan automáticamente un botón "Copiar código"
- El código debe ser limpio, sin texto extra que no sea código
- Esto facilita que Justin pueda copiar el código fácilmente sin editar
